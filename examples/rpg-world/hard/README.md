## How each original failure shape was retired

1. **Deep dependency DAGs** (`tech-*`) â€” the numeric-threshold **landmark term** +
   the **`Numeric` demand tier** (default since v0.2) restore the gradient the
   delete-relaxation loses when contributions converge.
2. **Multi-path numeric goals** (`mint-fortune`) â€” same demand tier: cumulative
   availability doesn't dither between interchangeable producers.
3. **Cyclic resource regeneration** (`farmstead-big`, `bread-line`) â€” demand
   survives consumption across rounds; pruning keeps the search out of the
   irrelevant-accumulator swamp.
4. **The village shape** (`gather-build`) â€” **goal-relevance pruning**, default-on
   since v0.2.2 (previously behind `FF_TDEMAND`).
5. **Big conjunctive goals** (`order-8/12`) â€” still the border. The architecture's
   answer is the **decomposer**: `ff --decompose` splits the order into per-part
   contracts and stitches a validated plan. `order-5` solves whole; 8+ wants
   splitting.

## What's robust

- **Deep travel is fine.** A 20-hop corridor solves â€” `travel` gives the heuristic a
  clean distance gradient, so depth alone isn't the enemy.
- **Tight allocation is fine.** `log-scarcity` correctly splits 4 logs (saw 2 +
  burn 2) rather than greedily blowing them on one product.
- (From the broader suite: deep *linear* accumulation solves too â€” `deepmine` is 15
