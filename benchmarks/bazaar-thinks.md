# Bazaar thinks — the game track's scoreboard

Generated by `cargo run --release -p ferroplan --example bazaar_thinks`
(0.13 Phase 3). Fixtures: `benchmarks/bench/bazaar-chain*.pddl` — 12
holders x 40 items, wants-gated barter; goal depth k forces a k-hop
trade-up chain (every trade crosses one want-edge, so ANY plan spends
>= k trades per chain; junk picks strand the chain item). The world
grounds ONCE per fixture; each cell below is an independent forked
mind: `fork` + `set_goal` + ONE bounded think (`replan_budgeted(B,
256 MB)`) — eval budgets, so thinks are deterministic at any thread
count. Cell = `plan-length @ evals-spent`, `--` = honest
budget-exhausted verdict. Whatever the curves say, they ship.

## Solo chains: heuristic-transparent (a finding, not a failure)

A single trade-up chain is EXACTLY visible to the relaxed-plan
heuristic (relaxation never spends the offered item, but a solo
chain never needs it back), so every depth solves at k+1 evals in
well under a millisecond — an NPC can chase an 11-hop trade chain
EVERY TICK. That is the game-track headline.

## Contended chains: where the curves live

`bazaar-chain-x2`: TWO chains cross at every vendor (v-j holds A-j
and B-j, wants A-(j-1) and B-(j-1)), both fed from one mind's two
seeds; goal k wants BOTH depth-k items (>= 2k trades). The
relaxation cannot see the contention — one offered item "advances"
both chains at once in relaxed space — so real search must dodge
cross-picks that strand a chain. Budget-exhausted verdicts appear
here.

## Per-mind retained memory and world load

- shared world payload (once, however many minds): ~0.1 MB solo / ~0.2 MB contended
- private state per forked mind: ~0.05 KB
- world load: 0.03 s solo / 0.07 s contended, once; forks are O(state-copy) — see the `many_minds` example for the 12-NPC load story
